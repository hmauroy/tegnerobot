Efficient Path Filling Algorithm Demo

This demo shows an efficient implementation of the even-odd fill rule for SVG paths, using techniques similar to those in browser rendering engines:

  • Active Edge Table (AET): Tracks only edges that cross the current scanline
  • Edge Buckets: Pre-sorts edges by their starting Y-coordinate
  • Analytical Intersection: Calculates exact intersection points mathematically
  • Scanline Algorithm: Processes the image line by line with optimal data structures

How It Works

For each horizontal scanline from top to bottom:

  1. Add new edges that start at this scanline height (from edge buckets)
  2. Remove edges that end before this scanline
  3. Calculate x-coordinates where each active edge intersects the scanline
  4. Sort intersections from left to right
  5. Fill between pairs of intersections (even-odd rule)

The algorithm achieves efficiency by:

  • Only processing edges that could potentially intersect the current scanline
  • Pre-sorting edges into "buckets" by their starting y-coordinate
  • Using mathematical interpolation rather than sampling for intersections
  • Maintaining sorted active edges for faster processing
